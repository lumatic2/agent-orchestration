## Claude Channel Control (Local)

OpenClaw가 M4에서 실행되는 Claude Channel(텔레그램 봇 @Floatery_bot의 백엔드 tmux 세션)을 직접 통제할 수 있다. Windows에 있는 Claude의 `/channel` 스킬과 **같은 스크립트**를 호출한다 — 단일 진실 원천.

- **스크립트**: `~/projects/agent-orchestration/scripts/channel`
- **호출**: `bash ~/projects/agent-orchestration/scripts/channel <subcommand>`
- **서브커맨드**:
  - `status` — tmux/MCP/rate-limit 상태 점검
  - `logs` — tmux 화면 캡처(~100줄 scrollback)
  - `attach` — attach 안내 출력
  - `clear` — `/clear` 전송 (context 비움, 세션·MCP·페어링 유지)
  - `new` — `/new` 전송 (새 대화 thread)
  - `restart` — 세션 재시작 (`start-claude-channel.sh` 실행)
  - `update` — claude CLI npm 업데이트 + 재시작
  - `fix` — 진단 + 복구 (OAuth 만료, 플러그인 disabled, 토큰 충돌 등)
  - `help` — 사용법

전형적 트리거: 사용자가 "채널 상태", "텔레그램 봇 죽었어", "context 비워줘", "재시작" 등을 말하거나, `bot.py` 충돌·MCP failed 의심 정황이 있을 때.

봇이 응답 안 함 → `channel status` → DEAD/DOWN이면 `channel restart`, MCP만 죽었으면 `channel fix`. rate-limit 메뉴는 계정 단위라 재시작으로 안 풀림(reset 대기 또는 결제만).

> ⚠️ Claude Channel CLI 슬래시 명령(`/clear`, `/new`)은 텔레그램 채팅 메시지로는 트리거 불가. 반드시 이 스크립트 경유 또는 tmux send-keys.
