# OpenClaw Worker Task Template

orchestrate.sh에서 OpenClaw로 작업 위임할 때 사용.

---

## 전체 템플릿 (여러 단계 GUI / 위험 작업)

```
[OpenClaw Worker Task]

목표:
- <이 작업의 최종 목적 1~2문장>

할 일:
1. <첫 번째 단계>
2. <두 번째 단계>
3. <세 번째 단계>

성공 조건:
- <무엇을 확인하면 완료인지>
- <최종 결과물 형태: 텍스트 보고 / 스크린샷 / 링크 / 상태값>

제약:
- 수정/삭제/전송/발송은 명시적으로 허용된 경우에만 수행
- 지정된 앱, 사이트, 계정 범위를 벗어나지 말 것
- 로그인/결제/게시는 별도 지시 없으면 하지 말 것
- <추가 금지 행동>

컨텍스트:
- Claude Code 역할: 판단, 계획, 코드 작업
- OpenClaw 역할: browser/GUI/computer-use 실행 담당
- <왜 이 작업을 하는지>
- <이미 Claude Code가 해둔 것>

보고 방식:
- 완료: 무엇을 했고, 결과가 무엇인지 3~6줄 요약
- 실패: 막힌 단계, 원인 추정, 필요한 추가 지시
```

---

## 축약형 템플릿 (단순 조회)

```
목표: <무엇을 할지>
할 일: <1>, <2>, <3>
성공: <완료 조건>
제약: <하지 말 것>
보고: <결과 형식>
```

---

## 사용 예시

```bash
# 단순 조회
bash orchestrate.sh openclaw "목표: 사이트 X의 오늘 매출 확인. 할 일: 대시보드 열기, 매출 찾기, 값 보고. 성공: 숫자 확인. 제약: 수정/전송 금지. 보고: 매출 숫자 3줄 요약." check-sales

# 복잡한 작업
bash orchestrate.sh openclaw-high "$(cat <<'EOF'
[OpenClaw Worker Task]
목표: ...
할 일: 1. 2. 3.
성공 조건: ...
제약: ...
컨텍스트: ...
보고 방식: ...
EOF
)" task-name
```

## 작업 유형별 선택

| 작업 유형 | 명령어 | 템플릿 |
|---|---|---|
| 단순 조회 | `openclaw` | 축약형 |
| 여러 단계 GUI | `openclaw` | 전체 템플릿 |
| 위험/복잡 작업 | `openclaw-high` | 전체 템플릿 + 제약 상세 |
