# /ui-ux-review — UI/UX 평가 프레임워크

웹사이트 또는 앱의 URL이나 설명을 입력받아 7개 기준으로 평가한다. 다음 순서대로 실행해라.

---

## STEP 0 — 평가 대상 수집

$ARGUMENTS에서 URL 또는 제품명을 파악한다. 이미 제공됐으면 STEP 1로 바로 진행.

없으면 AskUserQuestion 2회로 수집한다:

**질문 1** — AskUserQuestion을 호출해라:
- 질문: "평가 목적이 무엇인가요?" (header: "목적")
- A) 개선 우선순위 파악 — 어디를 먼저 손봐야 하는지
- B) 경쟁사 비교 — 다른 제품과 상대적 위치 파악
- C) 정기 모니터링 — 이전 평가 대비 변화 추적

**질문 2** — 첫 답변 후 AskUserQuestion을 호출해라:
- 질문: "이 제품의 주요 사용자는 누구인가요?" (header: "사용자")
- A) B2B 업무 사용자 — 실무자, PM, 설계사 등 전문직
- B) B2C 일반 소비자 — 비전문가, 다양한 연령대
- C) 개발자/테크 사용자 — API, 대시보드, 도구

두 답변 확보 후 STEP 1로 진행.

---

## STEP 1 — 7개 기준 스코어링 (각 1~5점)

각 기준을 1~5점으로 채점하고 근거 문구를 한 줄로 명시해라.

| # | 기준 | 정의 | 1점 | 3점 | 5점 |
|---|---|---|---|---|---|
| ① | **Useful** | 사용자의 실제 문제를 해결하는가 | 핵심 기능 없음 | 일부 해결 | 명확하게 해결 |
| ② | **Learnable** | 처음 쓰는 사람이 빠르게 익힐 수 있는가 | 설명 없이 불가 | 어느 정도 가능 | 설명 없이도 즉시 가능 |
| ③ | **Memorable** | 다시 방문했을 때 사용법을 기억하는가 | 매번 새로 학습 필요 | 일부 기억 가능 | 자연스럽게 기억됨 |
| ④ | **Effective** | 목표를 달성할 수 있는가 | 목표 달성 불가 | 부분 달성 | 완전히 달성 |
| ⑤ | **Efficient** | 목표까지 도달하는 단계가 최소화됐는가 | 불필요한 단계 다수 | 일부 최적화 | 최소 단계로 달성 |
| ⑥ | **Desirable** | 디자인·브랜드·감성이 신뢰를 주는가 | 불신감 또는 무관심 | 보통 | 신뢰·호감 유발 |
| ⑦ | **Delightful** | 예상을 넘는 긍정적 경험이 있는가 | 전혀 없음 | 미미하게 있음 | 인상적인 순간 존재 |

---

## STEP 2 — 종합 판정

총점(35점 만점) 기준으로 판정한다.

| 총점 | 판정 |
|---|---|
| 30점 이상 | ✅ 우수 — 유지·소폭 개선 |
| 20~29점 | 🟡 보통 — 우선순위 높은 2~3개 항목 개선 |
| 19점 이하 | ❌ 미흡 — 구조적 재설계 필요 |

---

## STEP 3 — 결과 출력

다음 형식으로 출력해라.

```
## UI/UX 평가 결과: [대상명]

**스코어링**
| 기준 | 점수 | 근거 |
|---|---|---|
| ① Useful | X점 | [한 줄 근거] |
| ② Learnable | X점 | [한 줄 근거] |
| ③ Memorable | X점 | [한 줄 근거] |
| ④ Effective | X점 | [한 줄 근거] |
| ⑤ Efficient | X점 | [한 줄 근거] |
| ⑥ Desirable | X점 | [한 줄 근거] |
| ⑦ Delightful | X점 | [한 줄 근거] |
| **합계** | **X / 35점** | |

**판정: [✅ 우수 / 🟡 보통 / ❌ 미흡]**

**가장 시급한 개선 항목 (상위 2개)**
1. [기준명]: [구체적 개선 방향]
2. [기준명]: [구체적 개선 방향]

**잘 되고 있는 부분**
- [강점 1~2개]
```
