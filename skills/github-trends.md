최신 GitHub 트렌드 리포트를 읽고 브리핑한다. 다음 순서대로 실행해라:

1. `~/projects/agent-orchestration/reports/` 에서 가장 최근 `github-trends-*.md` 파일을 찾아 읽어라.

2. 아래 형식으로 출력해라:

---
**GitHub 트렌드 브리핑 — [날짜]**

**즉시적용 항목**
| 레포 | 이유 | 적용 포인트 |
|---|---|---|
| [owner/repo](https://github.com/owner/repo) | ... | ... |

**참고 항목**
- [owner/repo](https://github.com/owner/repo) — 이유 한 줄
---

3. 즉시적용 항목 중 **지금 바로 적용하면 좋을 TOP 3** 추천:
   - 현재 오케스트레이션 시스템에 통합 가능한 것 우선
   - 설치/설정 복잡도가 낮은 것 우선
   - 각 항목: 왜 지금 좋은지 (2-3문장) + 적용 방법 간략 설명 + GitHub 링크

4. 마지막에 물어봐라: "어떤 레포 더 자세히 볼까요? 번호나 이름을 말해주면 README 열어서 분석해줄게요."

**[사용자가 특정 레포를 지정하면]**

5. `https://github.com/owner/repo` 를 WebFetch로 열어서:
   - 한 줄 요약 + 핵심 기능 목록
   - 설치 방법 (있으면 발췌)
   - 내 오케스트레이션 시스템에 어떻게 붙일 수 있는지 구체적 방안
   - 비슷한 대안 도구가 있으면 비교
