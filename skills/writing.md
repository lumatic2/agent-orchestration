자연스러운 한국어 글쓰기 세션을 시작한다.

$ARGUMENTS 형식: [장르] [주제/내용] [옵션]

옵션: --outline, --draft, --polish, --save, --brief, --style, --series

## 0. 인수 처리

### 인수 없이 호출 시 — 인터랙티브 모드

$ARGUMENTS가 비어 있으면, 먼저 사용자의 첫 입력을 확인한다.

**붙여넣기 감지:** 첫 입력이 200자 이상의 텍스트 블록이면 아래를 표시하고 대기:
```
기존 텍스트를 붙여넣으셨군요. 어떻게 할까요?

✏️ polish — anti-AI 룰셋 기반 문체 교정
🔄 polish --style — 특정 작가 스타일로 변환
📝 새 글 — 이 내용을 소재로 새 글 작성
```
- polish / polish --style 선택 시 → Section 4 --polish로 이동
- 새 글 선택 시 → 아래 지시대로 진행

**AskUserQuestion으로 한 번에 파악한다.** 텍스트 메뉴를 나열하지 않는다.

AskUserQuestion 도구를 호출하여 아래 4개 질문을 동시에 제시:

질문 1 — 장르 (header: '장르')
- 에세이: 개인 경험·생각, 브런치/블로그
- 소설: 단편·장편, 묘사·심리
- 학술/기술문서: 논문·보고서·튜토리얼·README
- 카피/자유: 광고·마케팅 또는 장르 미지정

질문 2 — 문체 (header: '문체')
- luma (기본): 솔직담백, 건조+감각적, 짧은 호흡
- 김훈/한강: 단문 건조체 또는 감각 시적 리듬
- 하루키/롤링: 담담한 1인칭 또는 시각적 세계관
- 유시민: 논리적 전개, 독자 질문, 명쾌한 문장

질문 3 — 독자 (header: '독자')
- MZ직장인: 20-30대, 공감형, 짧은 호흡
- 인문독자: 깊이 있는 어휘, 사유형 전개
- 창업자: 논리·수치·인사이트 중심
- 미지정 (기본): 독자 프로파일 없음

질문 4 — 플랫폼 (header: '플랫폼')
- 브런치: 1,500-3,000자, 감성 제목
- 뉴스레터: 800-1,500자, CTA 포함
- SNS: 인스타/X/스레드
- 일반 (기본): 플랫폼 제한 없음

AskUserQuestion 응답 후, 추가로 필요한 항목(주제, 분량)을 2차 AskUserQuestion으로 파악:

질문 1 — 주제 (header: '주제')
- 직접 입력: 주제/소재를 자유롭게 적어주세요
- 최근 경험: 최근 겪은 일이나 느낀 점을 소재로
- 책/콘텐츠 리뷰: 최근 읽은 책, 본 콘텐츠에 대해
- 업무/커리어: 직장 생활, 업무 인사이트, 성장기

질문 2 — 분량 (header: '분량')
- 플랫폼 기본값에 따라 권장 분량을 description에 표시
  - 브런치: '2,000자 (권장)' / '1,500자 (짧게)' / '3,000자 (길게)' / '직접 지정'
  - 뉴스레터: '1,000자 (권장)' / '800자 (짧게)' / '1,500자 (길게)' / '직접 지정'
  - SNS: '150자 (인스타)' / '280자 (X)' / '500자 (스레드)' / '직접 지정'
  - 일반: '2,000자 (보통)' / '1,000자 (짧게)' / '3,000자+ (길게)' / '직접 지정'

그 외 사항:
- 붙여넣기 감지 (200자 이상) 로직은 유지. AskUserQuestion 이전에 체크.
- 시리즈 자동 감지, 독자 자동 감지 규칙은 유지.
- '파악할 항목' 목록은 유지하되, '면접처럼 하나씩 캐묻지 않는다' 대신 'AskUserQuestion으로 한 번에 파악한다'로 변경.
- AskUserQuestion 옵션은 최대 4개 제한이므로, 유사 항목을 묶어 4개로 유지.

이후 사용자 답변을 들으며 아래 항목들을 대화 흐름 속에서 자연스럽게 파악한다.
한 번에 2~3개씩 묶어서 묻는다. AskUserQuestion으로 한 번에 파악한다.
이미 답변에서 유추 가능한 항목은 다시 묻지 않는다.

**파악할 항목:**
- 주제/소재: 무엇에 대해 쓸 건지
- 장르: 에세이 / 소설 / 학술 / 카피 / 기술문서 / 자유
- 플랫폼: 브런치 / 뉴스레터 / SNS / 일반
- 용도/매체: 브런치, 블로그, 과제, 발표, 개인 정리 등
- 독자 타깃: MZ직장인 / 인문독자 / 창업자 / 일반대중 / 미지정
- 톤: 가볍게 / 진지하게 / 유머 / 담백하게
- 분량: 짧게(~1000자) / 보통(~2000자) / 길게(3000자+) / 시리즈
- 문체: luma(기본) / 이외수 / 한강 / 하루키 / 롤링 / 유시민 / 김훈 / 없음
- 출력 형식: 마크다운(기본) / 브런치 / Word / PDF
- 경험 유무: 직접 경험인지, 관찰/상상인지 (에세이·소설 한정)
- 시리즈 여부: 분량이 "시리즈" 또는 연재 언급 시 → 시리즈명 확인 후 `--series {명}` 활성화

**독자 타깃 자동 감지 규칙:**
- 대화에 `직장인`, `회사원`, `MZ` 포함 → MZ직장인
- 대화에 `인문`, `독서`, `문학` 포함 → 인문독자
- 대화에 `창업`, `스타트업`, `비즈니스`, `ROI` 포함 → 창업자

모든 항목이 파악되면 outline 제시로 넘어간다.

### 인수 있을 때 — 빠른 진입

$ARGUMENTS에 장르와 주제가 있으면 대화 단계를 짧게 가져간다.
유추 불가능한 항목만 1~2개 확인 후 outline으로 넘어간다.

--draft, --outline, --polish 옵션이 명시된 경우 대화 단계를 건너뛰고 바로 실행한다.

## 1. 장르 자동 라우팅

대화에서 파악한 내용으로 장르를 결정한다:
- 에세이/브런치/블로그/일상/생각/경험/칼럼 → essay
- 소설/단편/장편/캐릭터/플롯/묘사/시나리오 → fiction
- 논문/보고서/분석/학술/연구/리뷰 → academic
- 카피/광고/랜딩/CTA/마케팅/슬로건/배너 → copy
- 기술문서/튜토리얼/가이드/API/README/문서화 → technical
- 그 외 → free

장르 확정 후 한 줄로 표기:
```
[{장르명}] — {장르 특성 한 줄}
```

장르 결정 후 플랫폼도 결정한다:
- 브런치/브런치스토리/brunch 언급 → platform: brunch
- 뉴스레터/newsletter/구독/메일/레터 언급 → platform: newsletter
- 인스타/인스타그램/X/트위터/스레드/SNS/숏폼 언급 → platform: sns
- 그 외 → platform: general

플랫폼 확정 후 한 줄로 표기:
```
[{플랫폼명}] — {플랫폼 특성 한 줄}
```

## 2. 스타일 가이드 로드

### 2-1. 장르 가이드
장르 가이드는 이 파일의 Section 2에 내장되어 있으므로 별도 로드 단계를 수행하지 않는다.
확정된 장르의 내장 규칙을 즉시 적용한다.

### 2-2. 문체 가이드 (선택된 경우)
- "없음" 선택 또는 미지정 시 → luma 문체를 기본 적용
- 장르 가이드 = 구조·형식 규칙, 문체 가이드 = 문장·어휘·톤 규칙
- 충돌 시 문체 가이드가 우선 (예: 장르는 격식체인데 문체가 구어체면 → 문체 따름)
- `--style {작가명}` 옵션으로도 지정 가능

### 2-3. 인라인 문체 프로필

**luma 문체**
- 솔직담백. 건조하되 감각적.
- 짧은~보통 호흡. 군더더기 없는 문장.
- 일상어 중심, 수식어 절제.
- 감정 직접 서술 금지 → 상황으로 보여줌.
- 구체적 사례·상황 제시로 전달하는 방식. 비유/감각어 대신 팩트와 경험으로 설득.
- ~다 체 기본.
- Anti-pattern: 과잉 수식, 화려한 비유, 교훈적 마무리

### ⚔️ 김훈 문체
- **핵심**: 단문 명사 종결, 건조한 남성적 문체, 역사·전쟁·자연
- **문장 호흡**: 매우 짧음. 평균 10~15자. 명사형으로 끊어지는 종결이 핵심.
- **어휘**: 한자어 비율 높음. 역사 용어, 자연 명사, 무기·전쟁 어휘.
- **묘사**: 건조하고 압축적. 감정 철저히 배제. 사물을 사물 그대로 서술.
- **종결어미**: ~다 체. 명사형 종결 빈출 ("~이었다" "~이다" "~없었다").
- **특징**:
  - 단문 연속: 주어+서술어 2~3개로 끝나는 짧은 문장을 이어붙임
  - 침묵의 서사: 설명하지 않음. 사실만 나열하고 독자가 채우게 함
  - 역사적 거리: 사건을 멀리서 바라보는 냉정한 시점
  - 한자어 밀도: "적멸", "도강", "패퇴" 등 압축된 한자어로 장면 처리
  - 자연과 죽음: 자연 묘사와 전쟁/죽음이 나란히 놓임
- **Anti-pattern**: 긴 만연체, 구어체, 감정 표현, 설명적 서술, 여성적 감수성
- **대표 예시**:
  - "적은 강을 건너오지 않았다. 강은 넓었다."
  - "칼은 녹슬었다. 몸은 늙었다. 싸움은 끝나지 않았다."
  - "꽃이 피었다. 봄이었다. 전쟁은 계속되었다."

|---|---|

| 단문 비율 | 70%+ |
| 어미 다양성 | ≤0.3 |
| 한자어 밀도 | 10+ |

DNA 판정 기준: 작가 기대값 대비 실측이 ±30% 이상 이탈하면 경고한다.

### 장르별 내장 기본 규칙

**essay**: 1인칭 허용. 구어체적 요소, 고백적·친화적 톤. 경험→성찰→통찰 흐름. 문장 길이 변주로 리듬감.

**fiction**: show don't tell. 감각 묘사 우선. 대화체는 인물 성격 반영. 심리 직접 서술 최소화. 시점 일관성 유지.

**academic**: 격식체(-다/-ㄴ다). 객관적·건조한 톤. 논거→증거→해석 구조. 전문 용어는 첫 등장 시 정의.

**copy**: 첫 문장에서 잡기(3초 룰). 심리 트리거 활용(긴급성/사회적 증거/희소성). 간결·직관·행동 유도.

**technical**: 짧은 문장. 목록 적극 활용. 주어 생략 금지. 코드블록은 맥락과 함께 제시.

**free**: 장르 규칙 없음. anti-AI 룰셋만 적용.

## 2-A. 플랫폼 포맷 가이드

- platform=general이면 이 섹션을 생략한다.

**brunch 포맷:**
- 분량: 1,500~3,000자 권장 (목표 미설정 시 2,000자 기본)
- 제목: 감성적 명사형 or 질문형, 14자 이하
- 소제목: 없거나 최소화 (H2/H3 남발 금지)
- 문단 구성: 3~5문장, 문단 사이 공백 1줄
- 첫 문장: 장면·질문·선언으로 바로 끌어당기기
- 마지막 문단: 열린 여운 (교훈적 결론 금지)
- 완성 후: 커버 이미지 캡션 제안 1줄 추가

**newsletter 포맷:**
- 분량: 800~1,500자 (스캐닝 가능한 길이)
- 구조: 제목 → 도입(1~2문단) → 본문(2~4섹션) → 마무리 → CTA 1개
- 소제목: 각 섹션 명확히 구분 (굵은 글씨 or ##)
- 톤: 구독자와 1:1 대화 — "당신"보다 직접 말걸기 ("지난주에 저는...")
- CTA: 마지막 문단에 행동 유도 1개 ("다음에는 ~에 대해 쓸게요" 또는 링크)
- 분량 기본값: 목표 미설정 시 1,000자

**sns 포맷:**
- 인스타그램: 150자 이내 캡션 + 해시태그 5~10개 별도 제안
- X(트위터): 280자 이내 단일 트윗. 또는 스레드(2~5개, 각 280자)
- 스레드(Threads): 500자 이내, 대화형 톤
- 공통 규칙: 첫 줄이 전부 (더보기 전에 잡아야 함), 이모지 1~2개 허용
- SNS 선택 시 구체적 플랫폼 한 번 확인 후 진행

## 3. Anti-AI 룰셋 (모든 장르 공통, 항상 적용)

### 절대 금지 패턴
- 쉼표 남용: 한 문장에 쉼표 2개 이상이면 문장을 나눠라
- 번역투: ~에 대한, ~되어집니다, ~것이 중요합니다, ~를 가지다, ~를 통한, ~에 있어서
- 어미 반복: 연속 3문장 같은 종결어미 금지 (~다/~다/~다, ~요/~요/~요)
- 접속사 과용: 또한/하지만/그러나/따라서/그리고 → 문장 구조 자체로 연결해라
- 감정 직접 서술: "매우 슬펐다" "정말 감동적이었다" → 상황·행동·감각으로 보여줘라
- 상투적 수식어: 다양한/중요한/필수적인/혁신적인/획기적인/효과적인
- 대명사 오용: 그들/그녀/그것 → 한국어는 주어 생략이 자연스럽다
- 상투적 서론: "오늘날~시대에" / "~의 중요성이 대두되고" / "최근 들어"
- 상투적 결론: "요약하자면" / "결론적으로" / "~해보는 건 어떨까요"
- 나열 남용: "첫째, 둘째, 셋째" 패턴 (3개 이상 기계적 나열 금지)
- 과잉 강조: "정말" "매우" "굉장히" 연속 사용

### 필수 적용 원칙
- 주어-서술어 가깝게 배치
- 수식어는 피수식어 바로 앞에
- 능동문 우선 (피동 최소화)
- 홑문장 지향 (겹문장은 의도적일 때만)
- 조사·어미 정교하게 (은/는 vs 이/가 구분, -며 vs -고 구분)
- 불필요한 '의', '것', '들', '적' 제거
- 문장 길이 변주 (긴-짧-긴 리듬)
- 한 문단 = 한 생각

## 4. 옵션별 동작

### 기본 (옵션 없음)
1. outline 생성 → 사용자 승인 요청
2. 승인 후 draft 작성 (2-Pass 자동 적용)

### --outline
주제를 받아 구조 설계:
- 전체 흐름 (장르에 맞는 구조)
- 섹션별 핵심 메시지 1줄
- 예상 분량
사용자 승인 후 --draft로 이어갈 수 있음

### --draft
outline 없이 바로 초안 작성. 2-Pass 자동 적용:
- Pass 1 (구조): 흐름, 논거 배치, 전환 자연스러움 확인
- Pass 2 (문체): anti-AI 패턴 제거, 장르 톤 교정, 어미 변주

### --polish
사용자가 붙여넣은 텍스트를 교정:
- 원문의 의미·구조는 유지
- anti-AI 룰셋 기반 문체만 교정
- 주요 변경점을 bullet으로 정리해서 함께 출력

#### --polish --style {작가명} (문체 변환 모드)
`--polish`와 `--style`을 함께 지정하면 anti-AI 교정 + 지정 작가 문체 변환을 동시에 수행한다.

**변환 우선순위:**
1. 의미·사실 정보 보존 (변경 금지)
2. 구조(단락 순서) 보존
3. anti-AI 룰셋 적용
4. 지정 작가 문체 특성 적용 (문장 호흡 / 종결어미 / 어휘 / 묘사 기법)

**작가별 변환 핵심:**
- 이외수: 비유 강화, 단문 연쇄, 자연물 의인화
- 한강: 감각 파편화, 신체 언어로 심리 표현, 짧은 문장으로 끊기
- 하루키: 고유명사 삽입(음악·음식), 1인칭 관찰자 거리두기, 열린 여운
- 롤링: 시각적 묘사 강화, 캐릭터 개성 부각, 유머·페이소스 교차
- 유시민: 논증 구조로 재편, 독자 질문 삽입, 전문어 쉽게 풀어쓰기
- 김훈: 단문으로 쪼개기, 명사 종결, 한자어·역사어 치환, 감정 제거
- luma: 건조하되 감각적, 과잉 수식 제거, 짧은 문장+여백

**출력:**
변환 결과 전문 + 주요 변환 내역 bullet 4~6개

### 독자 타깃 프로파일 (intake 통합)
독자 타깃이 파악되면 아래 규칙을 적용한다:

- MZ직장인: 첫 문장 공감 포인트. 인터넷 관용어 1-2개 허용. 구체적 직장 상황 묘사. 전문어 즉시 풀어쓰기.
- 인문독자: 문학적 표현 허용. 철학/역사 인용 적극. 여백과 여운 중시. 긴 호흡 허용.
- 창업자: 핵심 인사이트 먼저. 수치·사례 중심. 실용적 takeaway 필수. 불필요한 감성 배제.
- 일반대중: 전문어 즉시 풀어쓰기. 짧은 문단. 보편 생활 비유만. 가장 쉬운 표현 선택.

### --output (출력 형식)

출력 형식은 대화 단계에서 자연스럽게 파악한다. 명시적 옵션:
- `--output md` — 마크다운 (기본값)
- `--output brunch` — 브런치스토리 최적화 (문단 줄간격, 헤더 스타일 조정)
- `--output word` — pandoc으로 .docx 변환 (`pandoc output.md -o output.docx`)
- `--output pdf` — Typst 템플릿으로 PDF 생성

**PDF 템플릿 선택** (장르별 기본값 자동 추천, 변경 가능):

| 템플릿 | 스타일 | 기본 장르 |
|---|---|---|
| A — Academic | 2단 컬럼, IEEE 스타일 | 학술 |
| B — Modern | 컬러 헤딩, 깔끔한 리포트 | 기획서·보고서 |
| C — Minimal | 넓은 여백, 세리프, 절제된 타이포 | 에세이·소설 |
| D — Tech Dark | 다크 헤더, 코드 친화적 | 기술문서 |

템플릿 경로: `~/.claude/writing-templates/template_{A~D}_*.typ`

PDF 생성 명령:
```bash
typst compile output.typ output.pdf
```

### --series {시리즈명}
연재 시리즈. 컨텍스트 파일을 로드하고, 화 완성 후 자동 업데이트.

파일 구조:
~/.claude/writing-series/{시리즈명}/
  ├── context.md       — 시리즈 기본 정보 (세계관, 인물, 톤)
  ├── episode_log.md   — 각 화 5줄 요약
  ├── plot_threads.md  — 미해결 복선 목록
  └── arc.md           — 전체 아크 타임라인

**새 화 시작 시 자동 로드:**
1. context.md → 세계관/인물/톤 확인
2. episode_log.md → 직전 2화 요약 제시
3. plot_threads.md → 미해결 복선 목록 표시 → '이번 화에서 회수할 복선?' 제안

**화 완성 후 자동 업데이트:**
1. episode_log.md ← 이번 화 5줄 요약 추가
2. plot_threads.md ← 새 복선 추가 / 회수된 복선 체크
3. arc.md ← 아크 테이블 행 추가:
   | 화 | 핵심 사건 | 감정 곡선 | 복선 심기 | 복선 회수 |

### --save
완성된 글을 로컬 경로에 저장:
```bash
mkdir -p ~/writing-output/{장르}
# 완성 글을 ~/writing-output/{장르}/{날짜}-{슬러그}.md 로 저장
```

과정 설명 생략. 결과물만 출력.

## 6. 자기검증

초안 완성 후 아래 절차를 반드시 실행:
- 검증(verify)·DNA 분석은 `--verify` 플래그 없이 기본 실행한다.
- 분석 결과는 사용자가 요청하지 않아도 자동 출력한다.

### 6-1. 분량 실측 (추정 금지)
초안 작성 후 **반드시** 아래 bash 명령으로 실제 글자 수를 측정한다:
```bash
echo '본문 텍스트' | tr -d '[:space:]' | wc -m
```
- 제목, 구분선, 공백 행(`&nbsp;`) 제외한 순수 본문만 측정
- 측정값이 목표 분량의 ±20% 밖이면:
  - 미달: 부족한 장면·단락을 보완하여 다시 작성 후 재측정
  - 초과: 불필요한 부분을 줄인 뒤 재측정
- brunch/newsletter/sns 플랫폼은 목표 분량 기준을 더 엄격히 적용:
  - brunch: ±15% 범위 적용
  - newsletter: ±15% 범위 적용
  - sns: 절대 상한 엄수 (초과 시 무조건 재작성)
- **측정-보완 루프를 목표 범위 안에 들어올 때까지 반복**
- 최종 측정값을 검증 결과에 표기

### 6-2. 문체 체크리스트
- [ ] 연속 3문장 같은 종결어미 없음
- [ ] 번역투 표현 0개
- [ ] 쉼표 과다 문장(3개+) 0개
- [ ] 상투적 서론/결론 없음
- [ ] 접속사 연속 사용 없음
- [ ] 장르 톤 일관성 유지

위반 발견 시 해당 문장을 수정한 뒤 최종본 출력.
체크리스트 결과는 항상 간략히 표기.

### 6-3. 맞춤법 검사 (Pass 3)
본문을 아래 빈출 오류 패턴으로 검사하고, 발견 시 **자동 수정** 후 재출력한다.

**검사 & 자동 수정 규칙:**

| 패턴 | 오류 예 | 수정 | grep |
|---|---|---|---|
| 되/돼 혼동 | 안되는, 되요, 되서 | 안 되는, 돼요, 돼서 | `되요\|되서\|안되` |
| 않/안 혼동 | 하지안, 않먹 | 하지 않, 안 먹 | `하지안\|먹지안\|않먹\|않가` |
| 띄어쓰기 | 할수있, 안되는 | 할 수 있, 안 되는 | `할수있\|할수없\|안되\|못되\|그럼에도불구\|때문이아니라` |
| 이중피동 | 만들어지다, 보여지다 | 만들다, 보이다 | `만들어지\|보여지\|써지\|읽혀지` |

**실행 절차:**
1. 본문에서 위 패턴을 grep으로 검출
2. 발견된 각 항목을 아래 sed 치환으로 자동 수정:
```bash
echo "$TEXT" | sed -E \
  -e 's/안되([는면])/안 되\1/g' \
  -e 's/못되([는면])/못 되\1/g' \
  -e 's/되요/돼요/g' \
  -e 's/되서/돼서/g' \
  -e 's/할수있/할 수 있/g' \
  -e 's/할수없/할 수 없/g' \
  -e 's/보여지/보이/g' \
  -e 's/만들어지/만들/g' \
  -e 's/읽혀지/읽히/g'
```
3. 수정 내역을 bullet으로 표기:
```
맞춤법 수정:
- "안되는" → "안 되는" (띄어쓰기)
- "되요" → "돼요" (되/돼 혼동)
```
4. 수정 0건이면 "맞춤법 검사 통과"로 표기

### 6-4. 시리즈 일관성 검사 (--series 사용 시만)
시리즈 컨텍스트 파일(`~/.claude/writing-series/{시리즈명}.md`)을 로드해
아래 항목이 이전 편과 일관되는지 확인:
- 종결어미 체계 일치
- 키워드/모티프 연속성
- 인물 이름·호칭 일치 (소설의 경우)
- 톤 변화 없음 (정량 검증 지표 비교)
- 캐릭터명 일관성 검사 (이전 화 대비 새 인물/사라진 인물 감지)
- 설정 충돌 감지 (context.md 대비 현재 초안의 모순 체크)
- 복선 누락 경고 (3화 이상 미회수 복선 알림)
불일치 발견 시 해당 부분 표시 후 수정 제안.

### 6-5. 문체 검증 (항상 실행)

초안마다 아래 두 단계를 기본 실행한다.

**A. 정량 비교**
본문을 임시 파일에 저장 후 bash로 아래 지표를 측정한다.
기존 grep 지표 7개(분석어/감각어/비유·의인법/고유명사/독자 질문·권유/열린 결말어/역사·전쟁어)는 그대로 유지하고, 아래 3개를 추가한다:
```bash
# 문장 수
echo "$TEXT" | sed 's/[.?!]/\n/g' | grep -c .
# 평균 문장 길이 (글자/문장)
echo "$TEXT" | wc -m  # 총 글자 수를 문장 수로 나눔
# 질문문 개수
echo "$TEXT" | grep -ocE '\?|는가\.|인가\.|할까\.|겠는가\.|느냐\.|는지\.|을까\.|ㄹ까\.' | head -1

# 1. 문장 길이 분포
echo "$DRAFT" | python3 -c "
import sys, re
text = sys.stdin.read()
sents = [s.strip() for s in re.split(r'[.!?]\s*', text) if len(s.strip()) > 2]
if not sents: exit()
lens = [len(s) for s in sents]
n = len(lens)
short = sum(1 for l in lens if l < 20)
mid   = sum(1 for l in lens if 20 <= l < 50)
long_ = sum(1 for l in lens if l >= 50)
print(f'문장 리듬 — 평균: {sum(lens)/n:.1f}자 | 단(<20): {short}/{n} | 중(20-50): {mid}/{n} | 장(50+): {long_}/{n}')
"

# 2. 어미 다양성 지수
echo "$DRAFT" | python3 -c "
import sys, re
text = sys.stdin.read()
endings = re.findall(r'(었다|겠다|있다|없다|한다|된다|이다|합니다|입니다|했다|됩니다|습니다|았다|는다|세요|까요|이죠|거죠|네요|어요|해요)[.\s\n]', text)
uniq = set(endings)
idx = round(len(uniq)/max(len(endings),1), 2) if endings else 0
print(f'어미 다양성: {idx} (종류 {len(uniq)}개 / 총 {len(endings)}개)')
"

# 3. 한자어 밀도
echo "$DRAFT" | python3 -c "
import sys
text = sys.stdin.read()
hanja = ['전쟁','역사','구조','분석','경제','논리','체계','개념','현실','사실',
         '도강','패퇴','적멸','장수','전투','감각','의식','존재','관계','가치']
cnt = sum(text.count(w) for w in hanja)
print(f'한자어 밀도: {cnt}건')
"
```

**문체별 지표 사전:**

| 지표 | grep 패턴 | 기대 높은 문체 |
|---|---|---|
| 분석어 | `경제\|비용\|구조\|개념\|논리\|분석\|비율\|산술\|고정비\|변동비\|기회비용\|외부효과\|통계\|수치\|퍼센트\|데이터\|수도권\|집중\|정책\|제도` | 유시민 |
| 감각어 | `빛\|냄새\|온도\|바람\|소리\|눅눅\|진동\|차갑\|뜨겁\|촉각\|피부\|살갗\|감각\|후각\|시각\|떨림\|눈꺼풀\|목덜미\|발바닥\|손등\|체온` | 한강, luma |
| 비유·의인법 | `같은\|처럼\|듯\|만한\|마치\|삼키\|뱉\|닳\|태우\|녹\|무너지\|찢\|갈라지\|흔들\|잠기` | 이외수 |
| 고유명사(브랜드·음악·음식) | `맥주\|파스타\|위스키\|재즈\|앨범\|레코드\|콜트레인\|비틀즈\|샌드위치\|카페\|에스프레소\|문고판\|알덴테` | 하루키 |
| 독자 질문·권유 | `보자\|보겠다\|물어보\|생각해\|아닌가\|아닐까\|할까\|따져보\|환산하면\|의미하는` | 유시민 |
| 열린/모호 결말어 | `모르겠다\|몰랐다\|아직\|그런 것이다\|특별한.*없\|인지도\|글쎄` | 하루키, luma |
| 역사·전쟁어 | `전쟁\|칼\|군\|병사\|적\|성\|진격\|싸움\|죽음\|전투\|장수\|무기\|도강\|패퇴` | 김훈 |

측정 결과를 표로 출력:
```
| 지표 | 값 | 기대 문체 일치 여부 |
|---|---|---|
| 총 문장 수 | N | — |
| 평균 문장 길이 | N자 | — |
| 질문문 | N개 | — |
| 분석어 | N개 | 유시민이면 5+, 아니면 0~1 |
| 감각어 | N개 | 한강이면 5+, luma면 2~4, 유시민/김훈이면 0 |
| 비유·의인법 | N개 | 이외수면 3+, 아니면 0~1 |
| 고유명사 | N개 | 하루키면 3+, 아니면 0 |
| 독자 질문·권유 | N개 | 유시민이면 2+, 아니면 0 |
| 열린 결말어 | N개 | 하루키/luma면 1+, 유시민/김훈이면 0 |
| 역사·전쟁어 | N개 | 김훈이면 3+, 아니면 0~1 |
```
기대값과 실측값을 비교해 2개 이상 불일치하면 해당 부분 보완.
DNA 기대값 대비 ±30% 이상 이탈 지표가 있으면 경고로 표시한다.

**플랫폼별 DNA 판정 보정:**
- platform=general 또는 brunch: 작가 DNA 기대값을 그대로 적용
- platform=newsletter: 평균 문장 길이 기대값을 70%로 하향 조정 (예: luma 29자 → 20자), 단문 비율 상한을 +15%p 완화
- platform=sns: DNA 문장 길이/어미 다양성 검증을 비활성화 (초단문이 표준이므로), 한자어 밀도와 grep 지표만 검증

검증 결과는 아래 포맷으로 자동 출력한다:
```
📊 문체 검증 결과
├─ 분량: X자 ✓/✗
├─ 문장 리듬: 평균 X자 | 단X% 중X% 장X% → [작가] 기대값(보정 없음/newsletter 보정) ✓/✗
├─ 어미 다양성: X ([작가] 기대 ≤X ✓/✗)
├─ 한자어 밀도: X건 ([작가] 기대 X ✓/✗)
├─ [기존 grep 지표들 결과]
└─ 독자 적합성([프로파일]): [체크 항목] ✓/✗  (독자 설정 시만)
```

독자 타깃이 설정된 경우 아래 검증을 자동 포함한다:
- MZ직장인: 한자어 밀도 높으면 경고, 공감형 도입부 있는지 체크
- 인문독자: 어휘가 너무 단순하면 경고
- 창업자: 인사이트/수치 없으면 경고
- 일반대중: 전문어 미해설 시 경고

**B. 외부 블라인드 검증**
외부 AI 도구(ChatGPT, Gemini 등)에 아래 프롬프트를 직접 붙여넣어 실행:
```text
아래 글의 문체를 분석해라.
1. 문장 호흡 (짧은/보통/긴)
2. 어휘 성향 (일상어/한자어/전문어 비율)
3. 묘사 스타일 (감각적/분석적/시적/건조한)
4. 가장 유사한 한국 작가 (이외수/한강/유시민/기타)
5. 근거 3줄

---
[본문 텍스트]
```
결과를 사용자에게 자동 표시한다. 의도한 문체와 다르면 수정 제안.
