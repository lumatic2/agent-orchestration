# /music — Suno 음악 프롬프트 설계 세션

대화를 통해 원하는 음악을 구체화하고, Suno에서 바로 사용할 수 있는 프롬프트와 설정을 생성한다.

$ARGUMENTS 형식: [장르/분위기 설명] [옵션]

옵션: --quick (질문 없이 바로 프롬프트 생성), --lyrics (가사 포함), --instrumental (인스트루멘탈), --style [아티스트명] (가사 스타일 지정)

---

## 0. 인수 없이 호출 시 (인터랙티브 모드)

$ARGUMENTS가 비어 있으면:
```
어떤 음악을 만들고 싶으세요?

🎤 힙합
   └ 트랩 / 드릴 / 붐뱁 / 재즈힙합 / 로파이힙합

🎷 재즈
   └ 재즈힙합 / 퓨전재즈 / 스무스재즈 / 보사노바

🎸 록
   └ 인디 / 얼터너티브 / 팝록 / 포스트록

🎹 R&B · 소울
   └ 트랩소울 / 네오소울 / 펑크 / 컨템포러리 R&B

🎧 일렉트로닉
   └ EDM / 하우스 / 테크노 / 드럼앤베이스

🎻 클래식 · 오케스트라
   └ 미니멀 / 앰비언트 / 신포닉

🎵 K-POP · 팝
   └ 발라드 / 댄스팝 / 트로트

🌍 월드뮤직
   └ 라틴 / 아프로비트 / 레게

자유롭게 설명해주세요. 구체적일수록 더 정확한 프롬프트가 나옵니다.

이런 것들을 알려주시면 좋아요:
- **장르/분위기** — 재즈 힙합, 레이지 트랩, lo-fi R&B, 감성 발라드 등
- **BPM** — 느린(60~80), 보통(90~110), 빠른(120~140+)
- **메인 악기** — 어쿠스틱 기타, 808 베이스, 브라스, 로즈 피아노, 신스 패드 등
- **보컬 성별과 목소리 톤** — 남성 저음/허스키, 여성 청아함/파워풀 등
- **음역대** — 낮고 그루비하게, 중간 멜로딕, 높고 감성적으로 등
- **리듬** — 스윙, 붐뱁, 트랩 하이햇, 드럼 없이 베이스 중심 등
- **가사 스타일** — 빈지노 / 재키와이 / 타블로 / 블랙넛 / 던말릭 / 키드밀리 / pH-1 / lil boi
```

## 1. 음악 설계 대화 (--quick이 아닌 경우)

사용자의 초기 설명을 바탕으로 다음 요소를 순서대로 구체화한다.
이미 명시된 항목은 건너뛰고, 빠진 항목만 자연스럽게 질문한다.
한 번에 2~3개씩 묶어서 질문. 절대 전부 한꺼번에 묻지 않는다.

### 핵심 요소
| # | 요소 | 설명 | 예시 |
|---|---|---|---|
| ① | **장르** | 메인 장르 + 서브장르 | Jazz Hop, Trap Soul, Indie Rock |
| ② | **분위기/감정** | 곡 전체의 무드 | melancholic, euphoric, chill, dark, dreamy |
| ③ | **BPM** | 빠르기 | 70 (chill) ~ 140+ (energetic) |
| ④ | **악기 구성** | 핵심 악기 | piano, 808 bass, lo-fi guitar, synth pad |
| ⑤ | **보컬 스타일** | 보컬 특성 또는 인스트루멘탈 | male rap, female R&B vocal, no vocal |
| ⑥ | **레퍼런스** | 참고할 아티스트/곡 | "Nujabes 느낌", "The Weeknd 같은" |
| ⑦ | **곡 구조** | 원하는 전개 | intro→verse→chorus→bridge→outro |
| ⑧ | **언어** | 가사 언어 (가사 있을 경우) | 한국어, 영어, 혼합 |
| ⑨ | **길이** | 원하는 곡 길이 | 2분, 3분 30초 |

### 대화 가이드라인
- 음악 용어를 모르는 사용자도 편하게 대화할 수 있도록 쉽게 설명
- "비 오는 날 카페에서 듣는 느낌"처럼 감각적 표현을 음악 요소로 변환
- 장르 간 조합을 적극 제안 (예: "재즈 + 힙합 + lo-fi = jazz hop")
- 레퍼런스 아티스트를 언급하면 그 아티스트의 음악적 특징을 분석해서 반영

## 2. Suno v5 프롬프트 생성

설계가 완료되면 다음 형식으로 출력:

```
## 🎵 Suno 프롬프트 (v5)

### Style of Music (스타일 태그)
{장르, 분위기, 악기, 보컬 스타일을 Suno 태그 형식으로}

### Title
{곡 제목 제안 — 2~3개 후보}

### Suno 기본 설정
- **Mode**: Custom
- **Instrumental**: {Yes/No}
- **Version**: v5

### More Options (Suno v5)
- **Vocal Gender**: {Male / Female / 미지정}
- **Exclude Style**: {제외할 스타일. 프롬프트 안에 "no X" 형태로도 가능. 예: "screamo, heavy metal, autotune"}
- **Weirdness**: {Safe(0)~Chaos(100). 50이 "보통" 기준.}
- **Style Influence**: {Loose~Strong. 프롬프트 스타일에 얼마나 충실할지.}
- **Audio Influence**: {오디오 업로드 시에만 표시. 레퍼런스 오디오 반영 강도.}

### Persona / Inspo / Audio 탭
- **Persona**: {기존 곡에서 추출한 보컬 페르소나. 선택 시 Style of Music에 자동 반영. 없으면 "미사용"}
- **Inspo**: {내가 만든 곡(Suno로 생성한 곡)을 참고 파일로 업로드. 여러 곡 조합 가능. 없으면 "미사용"}
- **Audio**: {레퍼런스 오디오 업로드. 30초 스니펫 권장. Premier은 8분까지. 없으면 "미사용"}

### 프롬프트 (복사용)
\`\`\`
{Suno Style of Music 필드에 그대로 붙여넣을 수 있는 프롬프트}
\`\`\`
```

### Style of Music 프롬프트 작성 규칙 (v5)
- **글자 수**: v5에서 최대 1,000자 (v4 이전은 200자)
- **태그 우선순위**: Suno는 앞쪽 태그에 더 높은 가중치를 둔다. 순서: Genre → Mood → Instruments → Vocals → Production
- 쉼표로 구분된 태그 나열: 장르, 서브장르, 분위기, 악기, 보컬 스타일, BPM
- 구체적일수록 좋음: "hip-hop" 보다 "melodic rage hip-hop, dark synth leads, 808 bass, autotuned male rap"
- Suno v5는 자연어 설명도 잘 이해하므로 짧은 문장도 혼용 가능
- BPM은 숫자로 명시 가능: "140 BPM"
- **네거티브 프롬프트**: "no autotune, no heavy reverb, no falsetto" 식으로 제외 가능 (Exclude Style 필드와 별개)
- 레퍼런스 아티스트는 Style Influences 필드에 따로 넣는 게 v5에서 더 효과적
- 태그를 너무 많이 넣으면 "협상"이 됨 — 핵심 5~8개 권장

### Weirdness 장르별 권장값
| 장르 | Weirdness | Style Influence |
|---|---|---|
| Radio Pop / K-POP | 35~50 | 65~80 |
| Hip-hop / Trap | 40~55 | 60~75 |
| Worship / Gospel | 25~40 | 70~85 |
| Orchestral / Cinematic | 55~70 | 55~70 |
| Ambient / Experimental | 70~85 | 40~60 |
| 코러스 보호 (hook lock) | 25~40 | 70~85 |
| 브릿지 실험 | 55~70 | 45~65 |

### 슬라이더 사용 원칙
1. 슬라이더는 품질이 아니라 "행동"을 제어한다
2. 한 번에 하나의 슬라이더만 변경
3. 코러스는 보호 (낮은 Weirdness + 높은 Style Influence)
4. 실험은 브릿지에서
5. Audio Influence는 레퍼런스 업로드 시에만 사용 (60~75 권장)

### 가사가 있는 경우 (--lyrics 또는 대화에서 요청 시)

가사 섹션 추가:
```
### Lyrics (복사용)
\`\`\`
[Intro]
{가사}

[Verse 1]
{가사}

[Chorus]
{가사}

[Verse 2]
{가사}

[Bridge]
{가사}

[Outro]
{가사}
\`\`\`
```

가사 작성 규칙:
- Suno 구조 태그: [Intro], [Verse], [Chorus], [Pre-Chorus], [Bridge], [Outro], [Drop], [Hook], [Build]
- 보컬 지시 태그: [Male Vocal], [Female Vocal], [Whisper], [Spoken], [Belted], [Harmonies], [Ad-lib]
- 악기 브레이크: [Instrumental], [Instrumental Break], [Guitar Solo], [Piano Solo]
- v5 최대 곡 길이: 4분 (한 번 생성 기준). Extend로 연장 가능
- 가사 라인은 짧게 — 긴 문장은 보컬이 뭉개질 수 있음
- 한국어 가사는 자연스러운 운율과 라임 고려
- 영어 가사는 syllable count와 rhyme scheme 맞춤
- 발음이 어려운 고유명사는 발음 표기 권장 (garbling 방지)

### 가사 스타일 (--style 옵션)

`--style [아티스트명]`으로 지정하거나, 대화 중 "빈지노 스타일로 써줘" 등으로 요청 가능.
지정 시 아래 프로필을 기반으로 문체·라임·주제 의식을 해당 아티스트 결로 작성.

| 아티스트 | 핵심 특징 | 문체 | 주요 주제 |
|---|---|---|---|
| **빈지노** | 철학적·감성적, 자유로운 라임 구조 | 시적 이미지 + 일상어 자연스럽게 혼합, 영한 혼용 유려함 | 자아성찰, 예술과 일상, 도시 라이프, 자유 |
| **재키와이** | 직설적·감정 폭발, 날것의 에너지 | 짧고 임팩트 있는 라인, 감정을 날것으로 던짐 | 자기 확신, 여성 파워, 관계, 분노와 아름다움 |
| **타블로** | 문학적·철학적, 깊은 은유 | 복잡한 스토리텔링, 정교한 한영 혼합, 긴 호흡의 시 구조 | 삶과 죽음, 가족, 시간, 인간의 조건 |
| **블랙넛** | 자조적·아이러니, 유머와 진지함 공존 | 예상 못 한 펀치라인, 솔직한 자기 묘사, 도발적 유머 | 사회 비판, 일상의 불편한 진실, 자기 비하 |
| **던말릭** | 에너지·직관적, 압도적 플로우 | 짧고 강렬한 반복, 트렌디한 슬랭, 리듬 우선 | 자신감, 현재에 집중, 바이브 |
| **키드밀리** | 냉소적·스웨거, 날카로운 관찰 | 건조한 위트, 의외의 비유, 멈칫하게 만드는 한 줄 | 힙합 씬 비판, 자기 세계, 아이러니 |
| **pH-1** | 감성·서정적, 멜로디컬한 래핑 | 부드러운 한영 전환, 감정선이 뚜렷한 서사 | 관계, 그리움, 정체성, 성장 |
| **lil boi** | 그루비·세련됨, 재즈·소울 감성 | 여유 있는 플로우, 감각적 묘사, 도시적 분위기 | 여유, 멋, 도시의 밤, 자기표현 |

**스타일 미지정 시**: 장르와 분위기에 맞는 최적의 스타일을 자율 판단해 작성.

### 자연스럽고 감성적인 가사 원칙

**① 감정을 직접 말하지 않는다 (Show, Don't Tell)**
- "슬프다" → "창문에 빗소리만 들려"
- "행복하다" → "괜히 혼자 웃음이 나"
- 감정의 이름 대신 그 감정을 느낄 때의 장면/감각을 묘사

**② 구체적인 이미지로 공감을 만든다**
- "좋은 하루" → "단골 빵집 아저씨 미소"
- 누구나 공감할 수 있는 작고 구체적인 디테일이 보편적 감동을 만듦
- 시각 · 청각 · 촉각 · 냄새 등 감각 묘사 적극 활용

**③ 일상 언어 + 시적 표현을 섞는다**
- 너무 문학적이면 래핑이 어색해지고, 너무 평범하면 밋밋함
- 평범한 말 3줄 → 갑자기 한 줄 시 → 이 리듬이 감성을 만듦
- 긱스, Sik-K, pH-1 가사 참고: 일상어 속에 한 줄 시가 박힘

**④ 라임은 자연스럽게, 억지 금지**
- 라임을 위해 뜻을 비틀거나 어색한 단어 쓰는 것이 가장 티남
- 라임이 없으면 리듬감/반복으로 대체 가능
- 완벽한 라임보다 흐름이 우선

**⑤ 한 구절에 아이디어 하나**
- 여러 감정/메시지를 한 번에 넣으면 산만해짐
- Verse 1 = 상황 설정, Verse 2 = 심화/전환, Bridge = 클라이맥스 전 여백
- 각 섹션마다 역할 명확히

**⑥ 후렴(Chorus)은 단순하고 반복 가능하게**
- 들었을 때 바로 따라 부를 수 있는 한 문장이 핵심
- 감정의 정수를 압축 — 길고 복잡한 후렴은 기억에 안 남
- 한국어+영어 혼합 시 후렴의 영어는 감각적 짧은 단어로 (punch word)

## 3. --quick 모드

질문 없이 사용자의 설명만으로 바로 프롬프트를 생성한다.
부족한 요소는 장르 관습에 맞게 자동 결정하고, 결정 근거를 간단히 표기.

## 4. 프롬프트 개선 루프

출력 후 사용자가 수정 요청하면 반영해서 재생성.
- "좀 더 어둡게" → 분위기/악기 조정
- "BPM 올려줘" → 템포 변경
- "보컬 빼줘" → instrumental로 전환
- "가사 써줘" → 가사 추가

세션은 사용자가 만족할 때까지 유지.

## 5. Vault 참조

음악 허브의 장르별 특징과 Suno 프롬프트 템플릿 참조:
- `mcp__obsidian-vault__read_note` → Notion 음악 허브 장르 페이지의 Suno 실험 템플릿 활용
- 장르별 세부 특징은 Vault 10-knowledge 또는 음악 허브에서 조회

## 6. 세션 종료

프롬프트 확정 시:
```
✅ 프롬프트 완성! Suno에서 Custom 모드로 생성하세요.
스타일 태그와 가사(있으면)를 각각 복사해서 붙여넣으면 됩니다.
```
