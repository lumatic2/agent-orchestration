# [KNOWLEDGE] 경영전략·GTM·성장 프레임워크

## 비즈니스 모델 유형별 핵심 지표

### SaaS (B2B 구독)
| 단계 | 핵심 지표 | 벤치마크 |
|---|---|---|
| 성장 | ARR 성장률 | 초기: 3배/년, 스케일: 2배/년 |
| 효율 | CAC Payback | <12개월 (이상적 <6개월) |
| 유지 | NRR | >110% 우수, >120% 최우수 |
| 수익성 | Rule of 40 | >40 건전 |
| 영업 | ACV/AE 생산성 | $600K~$1M ARR/AE (국내 ~3~5억) |

### 마켓플레이스
| 지표 | 설명 | 목표 |
|---|---|---|
| GMV | 거래 총액 | 성장률 > Take Rate 하락률 |
| Take Rate | 수수료율 | 업종별 5~20% |
| Liquidity | 매칭 성공률 | 공급-수요 밸런스 |
| Repeat Rate | 재구매율 | >50% (건전한 마켓플레이스) |

### 커머스 (D2C·이커머스)
| 지표 | 벤치마크 |
|---|---|
| 전환율 | 1~3% (이커머스 평균) |
| 평균 주문금액(AOV) | 업종별 상이 |
| 재구매율 | >30% (D2C 건전) |
| 광고비 대비 매출(ROAS) | >3배 목표 |

## GTM (Go-to-Market) 전략

### GTM 모션 선택 기준
| 모션 | 적합 조건 | 대표 사례 |
|---|---|---|
| **Product-Led (PLG)** | ACV <$5K, 개인·팀 사용, 바이럴 | Slack, Notion, Figma |
| **Sales-Led (SLG)** | ACV >$20K, 기업 의사결정 복잡 | Salesforce, SAP |
| **Community-Led** | 개발자·크리에이터 타겟 | GitHub, Figma |
| **Marketing-Led** | 대량 인바운드 | HubSpot |
| **Channel-Led** | 파트너 네트워크 강점 | Microsoft |

### PLG (Product-Led Growth) 실행 체계
```
핵심 루프:
신규 유저 → 핵심 가치 경험(AHA Moment) → 습관 형성 → 확장 → 지인 초대

AHA Moment 설계:
- 가치 경험까지 시간 최소화 (TTV: Time to Value)
- Slack: 팀 내 2,000개 메시지 교환
- Dropbox: 파일 1개 저장 완료
- Figma: 팀원과 첫 협업 완료

PQL (Product Qualified Lead):
- 특정 행동 기준 충족 시 영업 개입
- 예: 무료 플랜 30일 이상 + 팀원 3명 초대 + 주 5회 이상 사용
```

### Korean GTM 특수 고려사항
- **네이버 SEO**: 구글 중심 SEO와 다름 — 블로그·카페 최적화 필수
- **카카오 채널**: B2C는 카카오 비즈메시지·알림톡이 이메일보다 효과적
- **네이버 스마트스토어**: D2C 브랜드 1차 채널 (커머스)
- **쿠팡 로켓그로스**: 빠른 성장이 목표면 쿠팡 풀필먼트 우선 검토
- **B2B 의사결정**: 한국 대기업 — 레퍼런스·벤더 등록·입찰 프로세스 고려

## 성장 단계별 전략 포커스

### 0→1 단계 (PMF 탐색)
**핵심 질문**: "누가, 왜 우리 제품을 씁니까?"

**실행 포커스:**
- ICP(Ideal Customer Profile) 극도 좁히기 — 처음 10명 직접 영업
- 정성 인터뷰 > 정량 설문 (문제 크기·빈도·현재 해결 방법 파악)
- Sean Ellis Test: "이 제품이 없어지면 얼마나 실망하시겠어요?" → 40% "매우 실망" = PMF
- MVP 범위: 핵심 가치 1개만 (나머지는 모두 제거)

**피해야 할 것**: 스케일링 투자, 브랜딩, 복잡한 기능 추가

### 1→10 단계 (성장 엔진 구축)
**핵심 질문**: "어떻게 반복 가능하게 성장하나?"

**실행 포커스:**
- GTM 모션 확정 (PLG vs SLG) + 채널 집중
- 영업 플레이북 표준화 (Win/Loss 분석)
- NRR 110%+ 달성 — 이탈 원인 구조적 해결
- CAC Payback 12개월 이내 달성 후 투자 확대

**핵심 지표**: ARR 성장률, NRR, CAC Payback

### 10→100 단계 (스케일)
**핵심 질문**: "어떻게 효율적으로 스케일하나?"

**실행 포커스:**
- 세그먼트 확장 (SMB → Mid-market → Enterprise 순서 일반적)
- 해외 진출 (글로벌 GTM 현지화)
- 조직 구조 전환 (기능별 → 사업부제 or 스쿼드)
- Rule of 40 유지하며 수익성 개선
- M&A 통한 제품·채널 확장

## 경쟁 분석 프레임워크

### VRIO 분석
| 자원/역량 | Valuable? | Rare? | Inimitable? | Organized? | 결론 |
|---|---|---|---|---|---|
| 자원 A | ✓ | ✓ | ✓ | ✓ | 지속 경쟁우위 |
| 자원 B | ✓ | ✓ | ✗ | — | 일시적 우위 |
| 자원 C | ✓ | ✗ | — | — | 경쟁 동등 |
| 자원 D | ✗ | — | — | — | 경쟁 열위 |

### 블루오션 4액션 프레임워크 (ERRC)
| 액션 | 질문 | 목적 |
|---|---|---|
| **제거(Eliminate)** | 당연하게 제공하지만 실제 가치 없는 것? | 비용 절감 |
| **감소(Reduce)** | 업계 표준 이하로 낮출 수 있는 것? | 비용 절감 |
| **증가(Raise)** | 업계 표준 이상으로 높여야 할 것? | 가치 증대 |
| **창조(Create)** | 업계에 없는 새 요소? | 차별화 |

### 포지셔닝 맵 설계
1. 구매 의사결정의 핵심 축 2개 선정 (예: 가격 vs 기능 깊이)
2. 경쟁사 포지션 맵핑
3. 빈 공간(White Space) 식별
4. 이동 가능성·자원 요건 검토

## 가격전략 프레임워크

### 가격 책정 3가지 접근
| 방법 | 기준 | 장단점 |
|---|---|---|
| **원가 가산** | 원가 + 목표 마진 | 단순하지만 가치 미반영 |
| **경쟁사 기반** | 경쟁사 대비 ±% | 빠르지만 차별화 포기 |
| **가치 기반** | 고객 WTP(지불 의향) | 이익 극대화, 조사 필요 |

### SaaS 가격 모델
| 모델 | 예시 | 적합 상황 |
|---|---|---|
| Per Seat | Slack, Zoom | 사용자 수 비례 가치 |
| Usage-based | AWS, Stripe | 사용량 비례 가치 |
| Flat + Tier | HubSpot | 기능 단계별 잠금 |
| Freemium | Notion, Figma | PLG, 바이럴 |

### 가격 인상 전략 (스타트업)
- 신규 고객 가격 인상 → 기존 고객 유지 (6~12개월 grandfather)
- 가격 인상 전: NPS·NRR 확인 (고객 만족도 충분해야)
- 패키지 재구성으로 인상 효과 → 기능 재묶기(Rebundling)

## OKR 실전 설계

### 좋은 OKR vs 나쁜 OKR
| 구분 | 나쁜 예 | 좋은 예 |
|---|---|---|
| Objective | "매출을 늘린다" | "한국 SaaS 시장에서 리더십을 확립한다" |
| KR | "고객을 더 늘린다" | "유료 전환 고객 500→800개 달성" |
| KR | "제품을 개선한다" | "NPS 42→55점 달성" |
| KR | "비용을 줄인다" | "CAC Payback 18개월→12개월 단축" |

### OKR 주기별 리듬
```
연간 OKR → 방향 설정 (CEO·C레벨)
분기 OKR → 핵심 집중 (팀·개인)
주간 Check-in → 진행률·장애물
분기 말 Review → 0~1.0 점수 + 학습
```

### OKR ↔ 보상 분리 원칙
- OKR 달성도를 직접 보상에 연동하면 → 안전한 목표 설정
- OKR은 학습·집중 도구 / 보상은 별도 기준으로

## 한국 스타트업 성장 사례 패턴

### 토스 (Toss) — 금융 슈퍼앱
- 0→1: 간편 송금 하나로 PMF 확인 → 핀테크 생태계 확장
- 성장 동력: 계좌 연동 데이터 → 서비스 개인화 → Lock-in
- 교훈: 단순한 핵심 가치 → 점진적 확장 (Never dilute core)

### 당근마켓 — 하이퍼로컬 마켓플레이스
- 0→1: 지역 거점 집중 (판교) → 밀도 확보 후 전국 확산
- 성장 동력: 지역 커뮤니티 네트워크 효과 → 광고 수익화
- 교훈: 마켓플레이스는 공급-수요 밀도가 핵심

### 무신사 — 패션 버티컬 커머스
- 0→1: 스트릿 패션 커뮤니티 → 쇼핑몰 자연스러운 전환
- 성장 동력: 콘텐츠(스타일링) → 구매 퍼널 직결
- 교훈: 커뮤니티 기반 커머스 = 낮은 CAC + 높은 LTV

## 재무 모델링 핵심 구조 (스타트업)

### 3-Statement Model 기본
```
손익계산서 → 현금흐름표 → 재무상태표 연결

매출 드라이버:
- SaaS: 고객 수 × ARPU
- 커머스: 방문자 × 전환율 × AOV
- 마켓플레이스: GMV × Take Rate

비용 구조:
- COGS: 매출원가 (서버·인프라·CS)
- S&M: 매출의 30~50% (성장기)
- R&D: 매출의 15~25%
- G&A: 매출의 10~15%
```

### Burn Rate·Runway 관리
```
Gross Burn = 월 총 현금 지출
Net Burn   = 월 총 지출 - 월 매출
Runway     = 현금 보유 / Net Burn

통상 VC 권장: 다음 라운드 클로즈까지 18개월 이상
경고: Runway < 9개월 → 즉시 조달 또는 비용 절감 액션
```

### Unit Economics 손익분기 계산
```
Contribution Margin = ARPU - 변동 COGS - CS 비용
CM이 양수여야 스케일링 의미 있음

CM이 음수인 경우:
→ 가격 인상 OR 원가 절감 OR 제품 가치 개선 먼저
→ 마케팅 투자로 해결 불가
```
