# Planby 영업 스크립트
> ICP: 중소 건설사 CEO (트랙 A — 자동화 아웃바운드)
> 업데이트: 2026-03-13

---

## 1. 첫 미팅 스크립트 (30분)

### 오프닝 (2분)
> "바쁘신 중에 시간 내주셔서 감사합니다. 오늘은 저희 소개보다 대표님 상황을 먼저 여쭤봐도 될까요?"

### 질문 먼저 — Discovery (10분)

핵심 질문 3개. 순서대로.

**Q1. 현재 AI 사용 현황**
> "요즘 업무에서 AI 쓰고 계신 부분이 있나요? 어떤 식으로 활용하고 계세요?"

→ 이미 쓰고 있다면: "어떤 한계를 느끼세요?"
→ 아직 없다면: "검토해보신 적은 있으신가요? 어떤 게 걸리셨어요?"

**Q2. 가장 시간·비용이 많이 드는 작업**
> "지금 프로젝트에서 반복적으로 시간이 많이 드는 작업이 어떤 게 있으세요? 도면 작업인지, 외주 CG인지, 제안서인지..."

→ 구체적인 Pain 확인. 숫자로 끌어내기.
> "그 작업 건당 보통 얼마나 걸리세요? 외주 주시면 비용이 어느 정도 나오나요?"

**Q3. 의사결정 구조 확인**
> "AI 도입 결정은 대표님이 직접 하시는 편인가요, 아니면 내부 검토 프로세스가 따로 있으신가요?"

---

### 사례 하나 — 상황에 맞게 선택 (5분)

Discovery에서 나온 Pain에 따라 하나만 꺼낸다.

**Pain이 CG·렌더링 비용이면**
> "비슷한 고민을 하셨던 분이 계셨는데요. 분양 시각화 작업을 외주에 맡기면 컷당 200~900만원에 4주가 걸렸거든요. 저희 커스텀 모델 붙이고 나서 같은 작업을 수 시간 안에 처리하고 계세요. 비용은 1/5 수준으로 줄었고요."

**Pain이 제안서·시각화 자료 제작이면**
> "글로벌 부동산 서비스사 사례인데요. 고객 제안서 하나 만드는 데 2~4주씩 걸리던 걸, AI 워크플로우 붙인 다음에 수 일로 줄였어요. 제안서 품질은 오히려 올라갔고 수주율도 개선됐고요."

**Pain이 설계 변경 검토 속도이면**
> "대형 건설사 AI 모델 구축 사례인데요. 설계 변경 검토가 평균 12일 걸리던 걸 수 일로 단축했어요. 건축 도메인 데이터를 학습시킨 커스텀 모델이어서 범용 AI로는 못 하는 수준이에요."

---

### 다음 행동 하나 — 클로징 (3분)

사례 전달 후 바로.

> "말씀 들어보니 [Q2에서 나온 Pain]이 딱 저희가 해결했던 케이스랑 비슷한 것 같아요.
>
> 저희가 보통 PoC를 먼저 제안드리는데요. 4~6주 안에 대표님 회사 데이터로 직접 돌려보고, 효과가 있으면 본계약으로 가는 구조예요. 비용은 1,500만원 선이고, 성공 기준을 처음에 같이 정하거든요.
>
> 다음 주 중에 구체적인 범위 가지고 한 번 더 뵐 수 있을까요?"

---

### 금지 사항
- 기능 나열 금지 ("저희는 이런 것도 되고 저런 것도 되고...")
- 사례 2개 이상 꺼내지 않기
- 가격 먼저 언급 금지 (Pain 확인 전)

## 2. 콜드 이메일 스크립트 (lemlist 트랙 A용)

> 대상: 중소 건설사 CEO
> 목표: 15~30분 미팅 1회 확보

### L1 — 첫 번째 이메일

**제목**: {{회사명}} 건축 AI 도입 관련 여쭤보고 싶은 게 있어서요

**본문**:
```
안녕하세요 {{이름}}님,

건설·건축 AI 전문 플랜바이의 유성입니다.

{{회사명}}처럼 [건설 시공 / 설계 / 개발] 하시는 분들이 요즘 AI 도입 고민을 많이 하시는데, 직접 구현하기엔 시간과 인력이 부족하다는 이야기를 자주 듣거든요.

한 가지만 여쭤봐도 될까요?
지금 업무에서 가장 반복적으로 시간이 드는 작업이 어떤 건가요?

15분만 이야기 나눠보고 싶습니다.
```

### L2 — 7일 후 팔로업

**제목**: Re: {{회사명}} 건축 AI 도입 관련

**본문**:
```
{{이름}}님, 앞서 메일 드렸던 플랜바이 유성입니다.

비슷한 규모 건설사 사례 하나 공유드리면 — 외주 CG 비용을 1/5로 줄이고 납기를 4주에서 수 시간으로 단축한 케이스가 있어요.

혹시 관심 있으시면 15분만 시간 내주실 수 있을까요?
```

### L3 — 14일 후 마지막

**제목**: 마지막으로 한 번만요

**본문**:
```
{{이름}}님, 마지막으로 한 번 더 드립니다.

타이밍이 안 맞으시면 괜찮아요. 나중에 건축 AI 도입 검토하실 때 생각나시면 연락 주세요.

유성 드림
planby.us
```

## 3. 사용 원칙

1. **미팅 전 고객사 DB 확인** — 기존 접촉 이력 있으면 해당 맥락 반영
2. **사례는 Pain 매칭** — Discovery 전에는 사례 꺼내지 않는다
3. **다음 행동은 하나** — PoC 미팅 일정 잡는 것 외 다른 액션 제안 금지
4. **트랙 B (대기업) 미팅은 별도 스크립트** — 이 문서는 중소 건설사 CEO 전용
