## 1. Was — 지난 4주간 무엇을 했는가

### 전략

AI 도구의 상품화(commoditization)가 가속화되고 있다. OpenAI·Google이 저가 범용 도구를 공급하면서 "AI 도구 구독료" 명분이 약해지고 있고, 국내외 건축 AI 경쟁사(Archigen·Archisketch·LookX AI·Veras)도 빠르게 등장하고 있다. 이 환경에서 기술 스펙으로 정면 경쟁하는 것은 소모전이다.

이에 대한 답으로 Veeva Systems의 플레이북을 참조했다. Veeva는 제약·임상 도메인에 특화된 생태계를 구축해 Salesforce 같은 범용 CRM이 침투하지 못하는 구간을 만들었다 — IPO 당시 밸류에이션 80x ARR. 플랜바이에 동일 원리를 적용했다.

**2026 전략 방향 및 전략문구 확정:**
- 핵심 축: **Custom Engine Year** — 커스텀 AI 엔진 수주·납품 역량 최대화
- 차별화 포지션: "더 좋은 AI"가 아닌 "더 잘 운영되는 AI 생태계"
- 확정 전략문구: *"플랜바이는 건축 AI를 단품으로 팔지 않습니다. 툴·워크플로우·교육이 결합된 통합 생태계로 장기 수익성과 재구매율을 높이는 기업입니다."*

이 문구가 회사소개서·홈페이지·영업 자료 전면 재작성의 기준점이 된다.

**딜 수주 판단 기준 설계 + Claude Code 스킬 구현:**
확정된 전략 방향에 따라 어떤 딜을 받아야 하는지 판단하는 기준을 설계했다. 이 기준을 Claude Code 스킬(`/decide-deal`)로 구현했다.

**인계 파일**: `decide-deal-installer.md` 하나만 본인의 Claude Code에 붙여넣으면 스킬이 자동 설치되고 사용법이 안내된다. 별도 설정 없이 즉시 사용 가능하다.

사용 예시:
```
/decide-deal 현대건설에서 AI 조감도 개발 문의 왔어. 예산 5천만, 기간 3개월.
→ ✅/🟡/❌ 판정 + CM 계산 + 최대 투입 인원 + 다음 액션 즉시 출력
```

### 재무

**성장 궤적 확보 — IR의 핵심 증거**
2024 매출 44,121,454원 → 2025 매출 289,086,455원 = **YoY +554% (6.5배 성장)**

매출이 +554% 성장하는 동안 영업비용은 +143% 증가에 그쳤다. 비용보다 매출이 4배 빠르게 성장하는 레버리지 구조가 데이터로 증명된다. 이것이 "팀 투자 해였다"는 주장을 숫자로 뒷받침한다.

2025년 말 기준: 현금 잔고 4.2억 / 런웨이 12~13개월 / Burn Multiple 1.64x / 재발주율 66.7% / Win rate ~63% — 제품이 실제로 가치를 만들고 있다는 증거다.

**Series A IR — 투자자 예상 질문 & 준비 상태**

착수 타이밍은 Phase 1(3~4월 내부 준비) → Phase 2(5~6월 네트워크 접촉) → Phase 3(7~8월 본격 IR)으로 설계했다. 문제는 타이밍이 아니라 투자자의 예상 질문에 얼마나 선제적으로 답을 준비하는가다.

### GTM

**ICP 세분화 — 아웃바운드와 Enterprise를 분리한 이유**

한 세그먼트로 두 가지 영업 방식을 동시에 운용하면 메시지가 희석된다. 트랙 A(중소 건설사 CEO)는 Clay + lemlist 자동화로 규모를 만들고, 트랙 B(건설 대기업 AI 임원)는 기존 레퍼런스와 인바운드로 깊이를 만든다. 두 트랙은 고객 규모·진입 방식·메시지가 모두 다르다 — 동시에 운영하되, 혼합하지 않는다.

**Clay 파일럿 실증 결과**

"아웃바운드를 자동화할 수 있다"는 가설을 수치로 검증했다.

- 리드 50명 수집 → 이메일 16명 확보 (**32% 전환**)
- 소요 시간: 30분 = 수동 대비 **5~10배 효율**

리드 모수 대비 이메일 주소 확보율 32%는 아웃바운드 자동화 파이프라인이 실제로 작동한다는 첫 번째 증거다. 3/16 16명 대상으로 lemlist 시퀀스를 발송했다 — 미팅 시점 기준 오픈율·클릭율 초기 데이터를 공유할 수 있다.

### 운영

**운영 자동화 — 같은 팀이 더 많은 프로젝트를 처리하게 만드는 것**

자동화는 단순 효율화가 아니다. 반복되는 내부 업무(재무 정산·인사 서류·정부지원 신청·계약 관리)에 사람이 묶이는 시간을 줄이면, 같은 팀이 더 많은 고객 프로젝트를 처리할 수 있다. Custom Engine Year 전략이 작동하려면 운영 자동화가 선행 조건이다.

**Slack AI 에이전트 (80% 완성)**

팀이 Slack 안에서 AI에게 질문하고, 문서를 요약하고, 반복 작업을 처리할 수 있는 내부 에이전트다. 현재 핵심 기능은 구현됐고 버그 안정화 단계다. 목요일 미팅 전 팀원 1명 실사용 스크린샷 + 사용 가이드까지 완성해 인계한다.

**블로그 자동화 — 추가 비용 0원**

Mac mini 스케줄러 + Gemini API로 콘텐츠를 자동 생성한다. 월·수·금 오전 9시 자동 발행, 주제 로테이션(주간뉴스 / 인사이트 / Q&A). 사람이 매번 쓰지 않아도 콘텐츠가 쌓인다 — 누적이 곧 인바운드 자산이 된다.

예시 글 2편을 이미 Notion 콘텐츠 DB에 적재했다. 홈페이지 완성 시 승인 버튼 하나로 발행되는 구조다.
- [인사이트] AI가 건설 현장 CG를 대체하기까지 — 비용·속도·품질 비교
- [인사이트] 분양 마케팅에서 3D 렌더링 vs AI 시각화 — 실무자가 알아야 할 차이

**유튜브 채널 기획 (방향 확정)**

블로그와 연동되는 영상 채널. 플랜바이 공식 계정, 목적은 B2B 인바운드 유입 + 브랜드 신뢰 구축. 형식은 5~8분 정보 영상.

포지셔닝: "건축·공간 AI를 실무에서 어떻게 쓰는가" — 기술 홍보가 아닌 실용 정보 채널. 콘텐츠 4기둥: 사례 소개 / 튜토리얼 / 트렌드 / 비교 분석. 블로그 인사이트 글을 영상으로 재활용하는 구조로 소재 이중 활용 가능. 첫 영상 후보: "AI가 건설 CG를 대체하기까지" (작성된 블로그 글 기반).

**신입 온보딩 패키지 v1 설계**

**프로젝트별 원가 추적 (Minimal Viable 방식)**

원가 추적의 목적은 세 가지다: ① 다음 재무제표에서 COGS를 합리적 근거로 계상 ② IR에서 Unit Economics("프로젝트 1건당 얼마나 남나?") 제시 ③ TIPS 정산 시 인건비 직접비 집행 근거 확보.

현실적으로 타임시트 도입은 팀 문화와 맞지 않는다. 대신 **FTE 기반 간이 추정** 방식을 사용한다.

> **FTE(Full-Time Equivalent)**: 풀타임 1명이 해당 프로젝트에만 100% 투입될 때 = 1 FTE. 절반 투입이면 0.5. 여러 명을 더하면 됨(예: 유상현 1.0 + 박현준 0.5 + 정지유 0.3 = 1.8 FTE).

운영 방식:
- **수주 전**: `/profitability` 스킬에 계약금액·기간·FTE 입력 → CM% 즉시 계산 → 수주 판단 근거
- **완료 후**: PM이 프로젝트 카드에 "실제 투입 FTE × 기간" 1줄 기록 → 실적 CM 사후 산출
- **외부 비용**: GPU·클라우드는 청구서에 이미 있음 / 외주비는 지급 시 적요에 프로젝트명만 추가

기억으로 추정이 어려우면 역산도 가능: 계약금액 기준 CM 60% 가정 → COGS 역산 → FTE 도출.

이 방법으로 쌓인 데이터가 세무사의 COGS 계정 신설 근거가 되고, Series A IR에서 Unit Economics 질문에 답할 수 있게 된다.
